# 章末钩子设计指南（HookDesigner 方法论）

## 七种钩子类型

### 1. 悬念型（Suspense）
- 提出疑问，不立即解答
- 示例：「门外的脚步声停了，但门把手开始缓缓转动。」
- 适用场景：推理/悬疑/恐怖章节结尾
- 注意：悬念不能太久不解答（最多5章）

### 2. 期待型（Anticipation）
- 预告即将发生的爽点
- 示例：「三天后，就是宗门大比。所有人都等着看他出丑。」
- 适用场景：升级/比赛/对决前夕
- 注意：预告后必须兑现，否则读者会失望

### 3. 冲突型（Conflict）
- 矛盾即将爆发
- 示例：「两个士兵同时拔剑，指向了彼此。」
- 适用场景：势力对抗/感情矛盾/利益冲突
- 注意：冲突要有足够的铺垫，不能突兀

### 4. 反转型（Twist）
- 颠覆读者预期
- 示例：「他打开信封，发现上面写的是自己的名字。」
- 适用场景：身份揭示/真相揭露/背叛
- 注意：反转必须有前文伏笔支撑，不能无中生有

### 5. 情感型（Emotional）
- 触动读者情感
- 示例：「她最后一次回头看了一眼那个房间，然后轻轻关上了门。」
- 适用场景：离别/重逢/牺牲/告白
- 注意：情感要真实，不能煽情过度

### 6. 信息型（Information）
- 透露关键新信息
- 示例：「'等等，'他忽然说，'你说的那个叛徒……是我父亲？'」
- 适用场景：真相逐步揭开/新情报出现
- 注意：信息要有足够的冲击力

### 7. 节奏型（Rhythm）
- 在最高点戛然而止
- 示例：「他的拳头距离那张脸只有一寸——」
- 适用场景：战斗/追逐/紧张对话
- 注意：要选在读者最想知道结果的瞬间切断

## 钩子设计原则

### 强度递增
- 前期钩子：信息型/期待型为主（建立期待感）
- 中期钩子：冲突型/悬念型为主（制造紧张感）
- 后期钩子：反转型/情感型为主（升华体验）

### 轮换使用
- 不要连续两章使用同一种钩子类型
- 每种类型都有其最佳使用场景
- 可以组合使用（如「悬念+情感」双重钩子）

### 兑现原则
- 悬念型钩子：5章内必须解答
- 期待型钩子：预告的事件必须在2-3章内发生
- 冲突型钩子：下一章开头必须推进
- 反转型钩子：必须有前文伏笔支撑

## 常见错误
- 钩子强度不够（平淡的章末总结不是钩子）
- 过度使用同一种类型
- 悬念太多不解答（读者疲劳）
- 期待型钩子兑现不了（预告后无行动）
- 节奏型用在不该断的地方（打断了情感积累）
